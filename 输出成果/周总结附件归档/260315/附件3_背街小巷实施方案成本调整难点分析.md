## 二、成本调整难点分析

### 难点一：成本核算颗粒度差异

#### 2.1.1 原版本问题

**核算颗粒度**：只到子项

**缺失内容**：
- 没有细化到每天每人做什么
- 没有明确人员配置
- 没有明确工作内容
- 没有明确成果输出

**问题表现**：
```
原版本：
体检部分 150万
├─ 调研前期 30万
├─ 调研 30万
├─ 调研后服务 30万
├─ 打印差旅 30万
└─ 平台开发 30万

问题：每个30万具体做什么？不清楚！
```

#### 2.1.2 新版本要求

**核算颗粒度**：细化到每天每人做什么

**需要明确的内容**：

1. **去北京调研**：
   - 去几个人？
   - 去几个部位？
   - 用半天还是一天？
   - 每个人承担什么任务？

2. **调研5天**：
   - 有几天在现场？
   - 平均到半天
   - 每天承担什么任务？

3. **调研后工作**：
   - 在办公室做哪些事？
   - 出哪些成果？
   - 哪些报告？

#### 2.1.3 工作量对比

| 项目 | 原版本 | 新版本 |
|------|--------|--------|
| 核算颗粒度 | 子项 | 每天每人 |
| 人员配置 | 不明确 | 明确到专家、高工、中级、初级 |
| 工作内容 | 不明确 | 明确到每天做什么 |
| 成果输出 | 不明确 | 明确到过程稿、过程成果 |
| 工作量 | 1天 | 3天（周三下午-周五） |

### 难点二：模块边界与内容重复

#### 2.2.1 实施方案章节模块

**已定模块**：
- 体检
- 导则
- 策略工具库

**约束条件**：
- 不能做大调整
- 每个模块内不能有其他模块的重复内容

#### 2.2.2 内容重复问题

**问题表现**：

```
体检模块
├─ 调研（重复）
├─ 现场勘察
├─ 数据分析
└─ 成果输出

导则模块
├─ 调研（重复）
├─ 标准制定
├─ 指导文件
└─ 成果输出

策略工具库模块
├─ 体检（重复）
├─ 调研（重复）
├─ 策略制定
└─ 工具开发
```

**重复内容**：
- 体检涉及调研
- 导则涉及调研
- 策略工具库也有体检和调研的部分

#### 2.2.3 解决方案

**核查方法**：
1. 逐项检查每个模块的内容
2. 识别重复内容
3. 明确重复内容的归属
4. 避免重复计算成本

**工作量**：
- 需要仔细核查
- 需要协调多个模块
- 需要反复确认

**砍价逻辑**：
- 按比例砍价
- 不考虑实际工作量
- 不考虑服务质量

**问题**：
- 砍价过度
- 影响服务质量
- 不利于项目推进

#### 2.5.3 共同目标

**设计咨询方和审计方共同服务于**：
- 真正的发包方
- 使用方

**目标**：
- 让成本有依据
- 科学客观合理

**本质**：
- 不应该存在虚报
- 服务应该有科学客观合理的成本

## 三、解决方案：逆向成本核算方法论

### 3.1 方法论概述

**传统方法**：
```
总成本 → 拆分到各模块 → 拆分到各子项
```

**逆向方法**：
```
单条街人员成本 → 整体算一下 → 拆出服务天数和服务人数
```

### 3.2 方法论步骤

#### 步骤1：先确定单条街的人员成本

**人员投入**：
- 1个专家
- 2个高工
- 1个中级
- 1个初级

**服务流程**：
- 完整服务流程
- 完美的落地呈现
- 完美的交付成果

**成本计算**：
```
单条街成本 = 人员成本 × 工作天数
           = (专家日薪 + 2×高工日薪 + 中级日薪 + 初级日薪) × 工作天数
```

#### 步骤2：整体算一下，稍微提高整体造价目标

**计算公式**：
```
整体成本 = 单条街成本 × 街道数量 × 调整系数
```

**调整系数**：
- 考虑管理成本
- 考虑风险成本
- 考虑利润空间

#### 步骤3：再拆出服务天数和服务人数

**需要明确**：
- 每项工作形成的过程稿、过程成果
- 每天这些人需要做什么

**输出内容**：
- 服务天数
- 服务人数
- 工作内容
- 成果输出

#### 步骤4：刚性成本

**兴业给到的单价**：
- 人员单价
- 差旅单价

**刚性成本**：
- 不能调整
- 必须按此核算

### 3.3 方法论优势

| 优势 | 说明 |
|------|------|
| 科学客观 | 基于实际工作量核算 |
| 有依据 | 每项成本都有出处 |
| 可追溯 | 可以追溯每项成本的计算过程 |
| 可验证 | 审计方可以验证成本合理性 |

## 五、经验总结

### 5.1 成功要素

1. **逆向思维**：从单条街成本反推整体成本
2. **精细化管理**：细化到每天每人做什么
3. **科学客观**：基于实际工作量核算
4. **有依据**：每项成本都有出处

### 5.2 改进方向

1. **建立成本核算标准**：形成标准化的成本核算方法
2. **建立工作量数据库**：积累各类项目的工作量数据
3. **优化核算工具**：开发成本核算工具，提高效率
4. **培训团队**：将成本核算方法推广给团队

## 六、AI协作过程（豆包）

### 6.1 协作工具

**使用工具**：豆包

**协作目标**：对体检部分的8个模块进行详细调整

### 6.2 遇到的问题

**问题一：模型幻觉**
- 如果全部在一个对话框里协作
- 写到后面可能就会有大量的模型幻觉
- 开始胡说八道

**问题二：操作繁琐**
- 经常出现来回切换窗口
- 复制粘贴
- 这种烦人的操作

### 6.3 解决方案：分模块协作

**协作策略**：
1. **第一个对话**：写一个（体检模块1）的主规则
2. **后续对话**：用规范说明书对2-8的清单、策略工具库的各个板块进行整理

**规则设置**：

1. **避免重复**：
   - 对于体检模块的2-8不允许有任务清单的重复
   - 同时也考虑一定不能出现关于导则编制的内容
   - 也不能出现后面策略工具库的内容

2. **负面清单**：
   - 关于城管委履职方面的一些关键词的负面清单
   - 不能出现，要换词

3. **约束条件**：
   - 单价的约束
   - 总价目标的约束
   - 子项、二级子项的拆分规则

### 6.4 输出成果

**规范说明书**：
- MD格式
- Word格式

**用途**：
- 对2-8的清单进行整理
- 对策略工具库的各个板块进行整理

### 6.5 工作目标

**核心目标**：
- 从和AI协作的场景上不要经常出现来回切换窗口、复制粘贴
- 这种烦人的操作

**优化方向**：
- 建立标准化的协作流程
- 形成可复用的规范说明书
- 提高协作效率
