### 2.1 成本核算规则包

#### 规则1：逆向成本核算方法论

**适用场景**：
- 成本核算
- 成本调整
- 成本审核

**规则内容**：
```
传统方法：总成本 → 拆分到各模块 → 拆分到各子项

逆向方法：单条街人员成本 → 整体算一下 → 拆出服务天数和服务人数
```

**步骤**：
1. **先确定单条街的人员成本**：
   - 人员投入：1个专家、2个高工、1个中级、1个初级
   - 服务流程：完整服务流程
   - 成果要求：完美的落地呈现、完美的交付成果

2. **整体算一下，稍微提高整体造价目标**：
   - 计算公式：整体成本 = 单条街成本 × 街道数量 × 调整系数
   - 调整系数：考虑管理成本、风险成本、利润空间

3. **再拆出服务天数和服务人数**：
   - 每项工作形成的过程稿、过程成果
   - 每天这些人需要做什么

4. **刚性成本**：
   - 兴业给到的人员单价
   - 差旅单价

**优势**：
- 科学客观：基于实际工作量核算
- 有依据：每项成本都有出处
- 可追溯：可以追溯每项成本的计算过程
- 可验证：审计方可以验证成本合理性

---

#### 规则2：成本核算颗粒度要求

**适用场景**：
- 成本核算
- 成本调整

**规则内容**：
```
原版本：只到子项
新版本：细化到每天每人做什么
```

**需要明确的内容**：
1. **调研工作**：
   - 去几个人？
   - 去几个部位？
   - 用半天还是一天？
   - 每个人承担什么任务？

2. **现场工作**：
   - 有几天在现场？
   - 平均到半天
   - 每天承担什么任务？

3. **办公室工作**：
   - 在办公室做哪些事？
   - 出哪些成果？
   - 哪些报告？

### 2.3 职责界面规则包

#### 规则7：第三方与甲方职责界面

**适用场景**：
- 方案编制
- 成本核算

**规则内容**：
```
第三方职责：
- 体系编制
- 技术支撑
- 专业服务

甲方职责：
- 主责主业
- 统筹协调
- 决策审批

关键要求：
- 第三方要分清界面
- 为城管委做好支撑工作
- 体系编制是支撑城管委的主责主业
- 遣词造句不能体现出做了主责主业的事
```

**风险点**：
| 风险 | 表现 | 后果 |
|------|------|------|
| 职责不清 | 第三方做了甲方的事 | 成本核算不合理 |
| 界面模糊 | 工作内容重叠 | 成本重复计算 |
| 遣词造句不当 | 文字表述越界 | 审核不通过 |
