# 项目规则

## 一、项目概述

**项目名称**: 打杂工具箱

**项目目标**: 通过AI辅助，实现工作总结的自动化、结构化和知识积累

**当前时间**: 2026年3月（模型应以此时为参照处理相对时间）

**核心功能**:
- 每日工作记录与深度探索
- 新闻监测与政策收集
- 周工作总结生成
- 项目跟踪与关联分析
- 知识库建设（项目知识 + 个人知识）

## 四、核心规则

### 1. 输入输出分离

| 区域 | 位置 | 说明 |
|------|------|------|
| 用户输入 | `工作记录\` | 用户放入工作碎片，AI只读 |
| 项目跟踪 | `项目库\` | 半自动，用户核查AI输出 |
| 个人知识 | `个人知识库\` | AI自动维护，用户可修正 |
| 工具定义 | `.trae\skills\` | AI定义，用户参考 |
| AI输出 | `输出成果\` | AI输出，用户可编辑 |

### 2. 知识萃取机制

| 来源 | 萃取内容 | 更新位置 |
|------|---------|---------|
| 每日工作 | 项目实体、人物、关联 | `项目库\_项目知识\` |
| 每日工作 | 个人技能、经验、学习 | `个人知识库\` |

### 3. 半成品机制

- AI生成初版（半成品），标注待完善内容
- 用户有时间再补充完善
- 降低用户负担

### 4. 以周为单位

- 每日记录碎片 + 知识萃取
- 周末生成周总结 + 知识汇总
- 周末归档

### 5. 文件命名规则

- 日总结（初版）：`YYYY-MM-DD-summary-draft.md`
- 日总结（最终版）：`YYYY-MM-DD-summary.md`
- 周工作总结：`MM.DD-MM.DD周工作总结.md`
- 新闻监测记录：`YYYY-MM-DD.md`
- 归档文件夹：`YYYY年第X周\`
- 对话日志：`对话日志_YYYY-MM-DD.md`

### 6. 对话日志存档规则

| 规则项 | 内容 |
|-------|------|
| **触发条件** | 每10轮对话（用户+AI为一轮）自动存档 |
| **存档方式** | AI自动存档 |
| **存档格式** | Markdown格式 |
| **文件命名** | `对话日志_YYYY-MM-DD.md`（当日多批汇总为一个文件） |
| **存档位置** | `输出成果\对话日志\YYYY-MM-DD\` |

**存档内容**：
- 完整对话记录
- 知识精华（核心洞察、经验总结、待解决问题、后续行动）
- 项目库更新
- 个人知识库更新

**定期汇总**：
- 日汇总：每日结束，提取当日知识精华
- 周汇总：每周末，提取本周核心经验
- 月汇总：每月末，生成月度知识精华报告

**AI职责**：
1. **对话计数**：跟踪对话轮次，在第10轮时触发存档
2. **自动存档**：达到10轮时自动创建存档文件
3. **内容提取**：自动提取知识精华（核心洞察、经验总结、待解决问题、后续行动）
4. **定期汇总**：日/周/月汇总
5. **知识更新**：更新项目库和个人知识库

**详细规则文档**：`输出成果\对话日志\存档规则.md`

## 八、关键要点

1. **输入输出分离**
   - `工作记录\` - 用户输入，AI只读
   - `项目库\` - 半自动，用户核查
   - `个人知识库\` - AI自动维护
   - `智能工具skills\` - 工具定义，AI定义
   - `输出成果\` - AI输出，用户可编辑

2. **知识萃取作为副产品**
   - 知识图谱在日常工作中自动生成
   - 无需专门维护

3. **项目知识 vs 个人知识**
   - 项目知识：项目绑定，客观事实
   - 个人知识：个人绑定，能力沉淀

4. **三个Skills协同**
   - **日工作记录**: 深度探索 + 知识萃取
   - **周工作总结**: 结构化总结 + 知识汇总
   - **新闻监测**: 自动化监测

5. **5个必需内容**
   确保周工作总结的完整性和规范性

## 十、知识积累与技能提取

### 1. Claudeception技能

**目的**：自动提取可复用的知识和经验，避免重复踩坑，实现持续学习。

**安装状态**：已全局安装到`~/.agents/skills/claudeception/`

---

### 2. 触发时机

在以下情况，主动使用Claudeception技能：

#### 2.1 代码开发场景

- **调试完成**
  - 发现非显而易见的解决方案
  - 通过试错找到变通方法
  - 解决根因不明显的错误

- **学习新模式**
  - 发现项目特定模式或配置
  - 学习框架特定用法
  - 掌握工具特定技巧

- **完成复杂任务**
  - 需要深入调查的问题
  - 涉及多个组件的集成
  - 需要创造性解决方案的任务

#### 2.2 平台开发场景

- **架构设计**
  - 发现新的架构模式
  - 解决系统集成问题
  - 优化性能瓶颈

- **工作流设计**
  - 发现新的工作流模式
  - 解决自动化问题
  - 优化流程效率

#### 2.3 项目谋划场景

- **方法论发现**
  - 发现新的谋划方法
  - 解决申报难题
  - 优化资金拼盘方案

- **政策解读**
  - 发现新的政策解读角度
  - 解决政策应用问题
  - 优化申报策略

### 3. 触发方式

- 输入：`/claudeception`
- 或输入："保存为技能"
- 或输入："提取技能"

---

### 4. 质量标准

Claudeception技能会自动评估是否提取：

#### 4.1 提取条件（必须全部满足）

- ✅ 需要实际发现（不是文档查询）
- ✅ 对未来任务有帮助（可复用）
- ✅ 触发条件清晰（明确何时使用）
- ✅ 已验证有效（确认可行）

#### 4.2 不提取的情况

- ❌ 只是文档查询
- ❌ 只适用于当前项目
- ❌ 未验证的解决方案

---

### 5. 技能格式

提取的技能会保存为Markdown文件，包含：

```yaml
---
name: [技能名称]
description: |
  [清晰描述，包含具体错误消息、框架版本、环境细节]
author: Claude Code
version: 1.0.0
date: [YYYY-MM-DD]
---

# [技能标题]

## Problem
[问题描述]

## Context / Trigger Conditions
[触发条件、错误消息、场景]

## Solution
[分步骤解决方案]

## Verification
[如何确认有效]
```

### 6. 最佳实践

#### 6.1 描述要具体

- ✅ "修复Prisma连接池错误"
- ❌ "数据库问题"

#### 6.2 命名要精确

- ✅ `prisma-connection-pool-exhaustion`
- ❌ `database-fix`

#### 6.3 步骤要清晰

- ✅ 分步骤，有验证
- ❌ 模糊描述，无验证
