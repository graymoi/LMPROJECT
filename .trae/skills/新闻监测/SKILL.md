- **版本**: 2.0
- **更新日期**: 2026-03-20

---

## V2.0 优化内容

### 优化1：增加转化机制

**问题**：收集的新闻难以转化为行动

**优化方案**：
```
新闻收集
    ↓
智能分析（与项目、政策关联）
    ↓
行动建议（是否需要跟进）
    ↓
输出报告（含行动建议）
```

### 优化2：增加关联分析

**问题**：与项目、政策关联度低

**优化方案**：
- 自动匹配关联项目
- 自动匹配关联政策
- 标注关联度（高/中/低）

### 优化3：增加优先级排序

**问题**：信息过载，缺乏重点

**优化方案**：
- 按关联度排序
- 按时效性排序
- 标注优先级（高/中/低）

### 优化4：增加行动建议

**问题**：收集后无行动

**优化方案**：
- 高优先级新闻：建议立即跟进
- 中优先级新闻：建议纳入周总结
- 低优先级新闻：仅记录

## 工作流程（V2.0）

```
步骤1：读取关键词库
─────────────────────
新闻关注关键词.md → 解析关键词列表

步骤2：执行搜索
─────────────────────
关键词列表 → WebSearch搜索 → 获取结果

步骤3：去重过滤
─────────────────────
搜索结果 → 读取history.json → 过滤已收集

步骤4：时间过滤
─────────────────────
过滤结果 → 只保留24小时内的新闻

步骤5：关联分析（新增）
─────────────────────
新闻内容 → 匹配项目/政策 → 标注关联度

步骤6：优先级排序（新增）
─────────────────────
关联度 + 时效性 → 计算优先级 → 排序

步骤7：生成行动建议（新增）
─────────────────────
高优先级新闻 → 生成行动建议

步骤8：生成记录
─────────────────────
最终结果 → 结构化输出
                    ↓
    输出成果\新闻监测记录\YYYY-MM-DD.md

步骤9：更新历史
─────────────────────
新收集的新闻 → 更新history.json
```

## 优先级计算规则

### 优先级计算公式

```
优先级 = 关联度权重 × 0.6 + 时效性权重 × 0.4

关联度权重：
- 高关联：1.0
- 中关联：0.6
- 低关联：0.3
- 无关联：0.1

时效性权重：
- 当天：1.0
- 1天内：0.8
- 2天内：0.6
- 3天内：0.4
- 3天以上：0.2
```

### 优先级划分

| 优先级 | 分数范围 | 处理方式 |
|--------|---------|---------|
| 高 | ≥ 0.7 | 立即跟进 |
| 中 | 0.4 - 0.7 | 纳入周总结 |
| 低 | < 0.4 | 仅记录 |

## 注意事项

1. **搜索频率**: 建议每日执行一次，避免频繁搜索
2. **关键词维护**: 定期更新关键词库，保持监测的准确性
3. **历史清理**: history.json 会自动清理30天前的记录
4. **行动跟进**: 高优先级新闻需要及时跟进，更新状态
5. **周报生成**: 每周五自动生成本周新闻汇总
